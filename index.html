Windows Internal

CHAPTER 2 System architecture

Hypervisor

VTL 0与VTL 1是一种类似隔离的机制，与原有的Ring机制有一定的独立性，VTL 0的内核无法访问VTL 1的用户态内存，虚拟机监视器是操作系统第一个初始化的组件，他会设置I/O MMU（避免通过DMA直接访问物理内存）以及SLAT，用来控制VTL 0中的内存访问。

VTL 1的secure kernel使用独立的二进制文件securekernel.exe，同时IUM的dll也有所不同，lumdll.dll对应ntdll.dll，lumbase.dll对应Kernelbase.dll

User Mode

subsystem

子系统有以下主要构件

  • 一个环境子系统进程实例Csrss.exe以及其加载的四个DLL，Basesrv.dll，Winsrv.dll，Sxssrv.dll，Csrsrv.dll
  • 位于内核态的Win32k.sys
  • 为控制台提供支持的Conhost.exe
  • 将多个可视窗口合成渲染到一个图形界面上的dwm.exe
  • 子系统DLL
  • 硬件的图形化设备驱动

  • 为什么要写这篇教程？

使用IntelliJ IDEA&Maven开发插件

一、前言

不知道各位开发插件使用的是什么IDE，eclipse? NetBeans?还是IDEA?相信各位应该对JetBrains全家桶有所耳闻，这也是(我自认为)目前最好的Java IDE，其拥有丰富的插件，具有极强的可扩展性，而且还有很多实用的功能，可以让程序员的生活更加美好。为了方便各位刚刚接触插件的开发者，节省他们因为不称手的工具而浪费的时间，我写下了这篇IDEA + Maven教程，因为，真的是太好用啦！

1. 注意事项

  • 本教程并不包含具体的Java语法教学，但理论上不会Java也能够看懂(但是看懂了也没啥用吧233)
    • Java在线学习：https://www.runoob.com/java/java-tutorial.html
  • 请不要与我争论哪一个 IDE 更好用，这没有意义，纯属浪费时间
  • 本教程的目标为教会各位如何使用 IDEA 配合 Maven 从头开始打包出一个可以使用的插件，其中会可能会穿插一些其他知识的教学，各位可以参考
  • 为了浅显易懂很多东西可能不会讲的很具体，部分细节也会简化，请不要杠谢谢
  • 本教程使用系统环境为Windows 10，Linux相关教程暂不提供，因为本教程绝大多数部分都是对于任意操作系统通用的；少部分(例如环境变量配置)，我相信有能力使用Linux进行开发的人一定可以自己去完成
