2. Programming a Guessing Game

本章节以一个猜数字程序为例介绍了各个方面的基础知识点，内容比较杂比较浅显

变量和引用

Rust中使用let关键字创建变量，默认情况下Rust的变量是不可变的，想要创建可变变量需要添加啊关键字mut

在给函数传参时使用&符号即可传递引用，默认情况下引用也是不可变的，需要使用mut关键字获取可变引用

疑问：如果以可变引用引用不可变变量会如何？
答：很明显会报错。。

基本错误处理

read_line会返回一个Result类型的值，Result值可能为Ok或者Err
如果想在遇到Err时不做处理简单的终止程序，可以调用Result的方法expect
如果对于一个可能出错的Result不做任何处理，编译器会发出警告

依赖引入与依赖版本

Cargo理解语义化版本规范，在0.x.y的开发阶段，x相同但y不同的依赖理应可以共用，因此cargo会使用x相同但y尽量大的依赖库
例如当前最新版本的rand为0.8.5，使用rand = "0.8.3"会引入0.8.5版本的rand

为了保证完全一致的特性，Cargo使用Cargo.lock来确保第一次build之后的所有build使用相同版本的依赖，例如

  • 在初始时cargo.lock为空，在编译时rand最新版本为0.8.5，配置文件指定版本0.8.3，则首次执行cargo build会拉取0.8.5版本的rand，cargo会将rand及其依赖的相关信息写入lock文件
  • 后续出现了版本0.8.6后，重新build此项目，cargo会检查lock文件，发现此项目首次编译使用0.8.5，便不会采用0.8.6版本，而是使用0.8.5版本编译
  • 如果需要强制使用最新修订号的依赖，需要使用cargo update指令

3. Common Pogramming Concepts

const关键字可以声明常量，与let的区别是

  • const可以在任何作用域使用，包括全局作用域
  • const的值只能设置为一个常量表达式，不能是在运行时才能确认的值
  • 可以对const执行可变引用，但是引用指向const的副本，不会影响原有值；对普通不变量进行可变引用无法通过编译

debug模式下整型溢出会导致panic，release不会

Expression & Statement

函数调用、大括号开始的新scope、macro调用都属于expression

  • 大括号expression的计算值是大括号里最后一行的值
    • 在大括号内最后一行如果是statement，则expression的值为()
  • expression打上分号将会转化为statement

if else结构其实也是一个expression，其值为实际执行的scope的值，因此可以用于let赋值

这种赋值要求if else每一个分支expression的返回类型必须相同

4. Understanding Owwnership

Ways Variables and Data Interact

Move

对于基本数据类型的传递，Rust与其他语言类似，直接复制值

x与y是两个独立的变量，指向栈上不同的空间

而对于引用类型的传递，Rust采用move的形势实现

在其它语言中，s2与s1内部的指针会指向同一片内存区域，而在Rust中，s1会直接失效，只有s2拥有该片内存区域的所有权

Clone

对于实现了Copy trait的类型（基本数据类型实现了），在复制时会自动进行深复制，同时复制前后的变量都不会失效

如果想让没有实现copy的引用类型执行深拷贝，需要调用clone方法

需要注意的是，函数参数的传递以及参数的返回都有类似的行为，实现了Copy的进行深复制，没有实现的会发生Move

References and Borrowing

为了简化代码，避免传参等情况导致的owner权限丢失，可以传递参数的引用

函数参数必须要声明其接受引用，调用时也必须使用&符号获取引用并传递

如果原变量是可变的，那么还可以获取其可变引用，但是如果同一个scope中已经有一个可变引用出现，则该变量不能再有其他的引用

准确来说是，如果一次可变借用发生了，那么在此次可变借用发生之前的任何借用不能再使用

这种借用是可以发生的，但是在mut_ref2之后就不能再使用inmut_ref和mut_ref1了

此外Rust编译器确保不会出现悬垂引用的情况

The Slice Type

String slice是一种对于某个String的特殊的引用，其类型为&str

6. Enums and Pattern Matching

rust中的枚举可以携带数据，并且统一枚举中不同枚举值可以携带的数据类型不一样

同时可以为枚举实现成员函数

Option Enum

Rust中有一类特别的枚举Option<T>

其有两种可能的值None和Some(T)

在某些可能返回缺省值的情况下，可以返回Option<T>类型

如果返回Some(T)则是有效值，返回None则为缺省值

Rust故意使Some(T)与T无法混用以便让开发人员注意处理返回None的情况

Match with Enum

match可以匹配枚举的值进行不同的操作，同时还可以取出其中可能携带的数据进行相应操作

当然match也可以用于处理Option枚举

match的_ => ()arm可以在其余情况下不执行任何操作

if let

if let可以针对当enum为某一特定情况时展开工作，在其他情况下执行else(如果存在)的操作，类似于一个简化版的match，不会显式要求处理所有情况

7. Managing Growing Projects with Packages, Crates, and Modules

Packages & Crates

crate是Rust编译器处理代码时的最小单位，crate分为binary crate和library crate

binary crate可以编译为可运行程序，library crate可以编译成库

package是一个或多个提供一系列功能的组合，一个package可以包含任意个bin crate，但只能包含一个lib crate

src/main.rs 是crate root

src/lib.rs 也是crate root

src/bin/里面的文件都会被当作独立的binary crate

[rust api guidelines](About - Rust API Guidelines (rust-lang.github.io))

Modules

crate root可以创建新的模块，在非根模块中也可以创建子模块

父模块不能直接访问子模块中的内容，除非子模块公开(pub)

同属一个父模块下的兄弟模块可以相互访问

根模块在路径中名称为crate

模块路径寻址支持相对路径和绝对路径，同时可以用super指代父模块

Best Practices for Packages with a Binary and a Library

在lib.rs中实现一套api，同时main.rs作为api的使用者实现功能

Structs公开时其内部成员依旧是private，需要单独指定pub成员；enum公开时其所有变体也都是公开的

通常使用use希望将某个函数引入scope时，不直接将其引入，而是引入其父模块，这样能够更好的区分从外引入的对象与本地对象；而引入结构体、枚举时则通常直接引入，这是一种习惯
