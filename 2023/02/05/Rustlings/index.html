Rustlings随记

看完了Rust入门书籍《The Rust Programming Language》，发现官方还提供了一个由小练习构成的项目rustlings，于是拿来练练手。大部分内容都比较简单，看过一遍书的话应该都能很容易做出来，这里只记录一些我觉得有意思的东西

VARIABLES

variables6.rs

使用const关键字声明constants常量时必须同时声明类型

PRIMITIVE_TYPES

这一章很多内容都可以通过rust的自动类型推断完成，不用显式声明

primitive_types3.rs

声明栈上定长数组的语法，不是很常用，容易忘

VECS

vecs1.rs

这里使用Vec::from可以直接把已有数组转为Vec

vecs2.rs

vec_map对v的迭代器使用map生成一个新的迭代器，并对新迭代器调用collect生成一个新的vec

MOVE_SEMANTICS

变量是某块内存区域的owner，变量是否可变只与变量本身有关，与内存区域无关，当发生move导致内存区域的owner变更时，如果旧owner不可变，可以让新的owner可变进而修改内存区域，e.g.

STRUCTS

structs2.rs

使用现有struct对象初始化新的struct对象，需要注意的是没有实现Copy trait的成员将会被move，同时旧struct会失效。本例中Order仅name字段为String，没有实现Copy trait，但是新的变量中给定了新的name，因此不会发生move，旧order_template仍可用。假如没有给定新的name，那么order_template中的name会move到新变量中，同时导致自身不可用

CONVERSIONS

try_from_into

上面这个是👴🏻本来的答案，但是hint这么说

Is there an implementation of TryFrom in the standard library that
can both do the required integer conversion and check the range of the input?

于是👴🏻对代码进行了修改，使用了try_into将i16转化为u8，并且使用map_err调整Result的类型以便使用?操作符，但是代码看起来更💩了是怎么回事呢，感觉不如上面那个好懂

另外在这里出现了一个问题，就是rust-analyzer和rustc对map中color的类型判断不同，rustc不过编译，在rust论坛上发帖问了一下找到了问题所在，是因为rustlings在编译时没有指定edition，默认edition是2018，但是rustlings生成的rust-analyzer配置文件指定edition为2021，这两个版本中[i16; 3]的into_iter行为不一样，具体原理我还没研究明白，这里先把论坛大佬的回答链接贴在这里，等我研究明白了再水一篇blog((

edit: 研究了一下，其实也没啥特别的，就是在早期Rust版本中，只有arrays的引用实现了IntoIterator，由于Rust自动引用与解引用的特性，调用array.into_iter()会解析为(&array).into_iter()，使用map获取到的类型自然成为了对array中存储类型的引用
